Event Creation Pipeline

flowchart TD
    A[Create Event] --> B[Step 1: Basic Info]
    B --> C[Auto Save Draft]
    C --> D[Step 2: Timeline]
    D --> E[Auto Save Draft]
    E --> F[Step 3: Rules]
    F --> G[Step 4: Quiz]

    G --> H{Quiz Required?}
    H -->|Yes| I[Attach Existing Quiz or Create New]
    H -->|No| J[Skip Quiz]

    I --> K[Step 5: Selection Criteria]
    J --> K

    K --> L[Step 6: Rewards]
    L --> M[Step 7: Notifications]
    M --> N[Step 8: Preview]

    N --> O{Validation Passed?}
    O -->|No| P[Show Missing Fields Checklist]
    O -->|Yes| Q[Publish Event]

    Q --> R{Already Published?}
    R -->|Yes| S[Show Unpublish Button]
    R -->|No| T[Published Successfully]

Counselling Booking Pipeline
Student Booking Flow
sequenceDiagram
    actor Student
    participant App as Flutter App
    participant VM as CounsellingViewModel
    participant API as FastAPI Backend
    participant DB as Database

    Student->>App: Opens Helping Support
    App->>VM: Load mentors + available slots
    VM->>API: GET mentors + slots in parallel
    API->>DB: Fetch active mentors and open slots
    DB-->>API: Data
    API-->>VM: Mentor list + slot list
    VM-->>App: Render recommended mentors first

    Student->>App: Selects mentor
    App->>App: Show available slots calendar view
    Student->>App: Taps slot
    App->>App: Show confirmation bottom sheet
    Student->>App: Confirms booking

    App->>VM: Optimistic booking state
    VM->>API: POST book slot
    API->>DB: Transaction check capacity
    DB-->>API: Booking saved
    API-->>VM: Session with Meet link
    VM-->>App: Show success screen
